# Тестовое задание BIOCAD (весна, 2026)
Данный репозиторий посвящён выполнению тестового задания на должность стажёра в группу вычислительной медицинской химии компани BIOCAD. Здесь представлено решение 2 задания: создание скрипта для конвертации молекул формата SDF/SMI в формат PDBQT для проведения молекулярного докинга в программе AutoDock Vina. Далее в документе представлена информация об основном содержании репозитория, краткая инструкция по запуску скриптов, коментарии к скриптам, где указаны различия в логике кода. 
# Структура репозитория
В представленном репозитории находяться неколько файлов с разным назначением. Папка **data** хранит входные файлы, данные которых используются для создания PDBQT файлов с помощью python скрипта. Блокноты Jupyter Notebook **ligand_prep_v1** и **ligand_prep_v2** содержат две версии скрипта для подготовки молекул. В папки **output** и **output_2** будут записываться выходные PDBQT файлы с соответствующих скриптов. В файле **requirements.txt** перечислены необоходимые библиотеки для корректной работы скрипта.
# Установка и запуск
Для запуска скрипта, вам необходимо сначала клонировать репозиторий и перейти в его корневую папку. Затем создайте отдельное окружение Python (рекомендуется Python 3.10) и установите зависимости, перечисленные в **requirements.txt** (например, используя conda: `conda install -c conda-forge rdkit meeko`). После этого запустите Jupyter Lab (или Jupyter Notebook) из того же окружения и откройте файл соотвествующего варианта скрипта. 

Для запуска первой версии скрипта просто выполните последовательно все ячейки ноутбука. При запуске второй версии, введите относительный путь файла, например, `data/example.sdf`. При необходимости можно заменить входной файл (по умолчанию запрашивается через `input()`). Результаты (PDBQT-файлы) будут сохранены в соответствующую папку output. Убедитесь, что входные файлы находятся в папке `data/`, или укажите полный путь к ним при запросе.
# Комментарии к коду
Я предоставил два варианта скрипта для конвертации лигандов в PDBQT: первый основан на пакетной обработке (загрузка всех молекул в список, затем подготовка и запись), второй — на итерационной (обработка каждой молекулы без промежуточного сохранения). Оба варианта корректно выполняют основную задачу: читают SDF или SMI, добавляют водороды, генерируют 3D-координаты и через Meeko создают PDBQT-файлы. 

Главное отличие — в подходе к управлению памятью. В первом варианте скрипт хранит все молекулы в списке, что удобно для отладки, но требует больше памяти при больших размерах входного файла. Второй вариант обрабатывает молекулы по одной, что экономит память и позволяет обрабатывать большие наборы данных. Дополнительное задание (гибкость алифатических циклов размером более 5) реализовано во втором варианте передачей параметра `min_ring_size=6` в `MoleculePreparation` (при этом параметр `rigid_macrocycles` по умоланию указан как `False` — гибкость включена). В первом варианте этот параметр отсутствует, но его можно добавить аналогично. 

Я рекомендую использовать второй (итерационный) вариант: он более экономичен, и позволяет быстро изменять входные файлы для конвертации (через указание пути файла в `input()`. В обоих вариантах скрипта я не стал проводить обработку ошибок по типу проверки `mol is None` или результата `EmbedMolecule`). Если вы хотите использовать другие входные файлы, отличные от представленных в папке `data/`, отдельно убедитесь, что они являються корректными и каждую молекулу можно конвертировать в PDBQT формат без ошибок.
